# Cabinet Maître Marie-Elise GBÈDO — Plateforme Digitale Premium

Une vitrine digitale d'exception pour le Cabinet Maître Marie-Elise GBÈDO, conçue avec une approche "Editorial Premium Magazine". Cette plateforme allie autorité juridique, élégance contemporaine et fluidité technologique pour offrir une expérience utilisateur hors du commun.

## Vision & Design

La plateforme repose sur une direction artistique millimétrée :
- **Typographie Sophistiquée** : Contraste entre la noblesse de la *Cormorant Garamond* (Serif) pour les titres et la modernité de la *Montserrat* (Sans-Serif) pour le corps de texte.
- **Palette de Couleurs Statutaire** : 
  - **Bordeaux Vibrant (#800020)** : Symbole d'autorité, de passion et d'engagement.
  - **Gris Anthracite (#2F2F2F)** : Pour la structure, la sobriété et le prestige.
  - **Blanc Pur (#FFFFFF)** : Pour la clarté, l'espace et la respiration éditoriale.
- **Expérience Immersive** : Curseur interactif personnalisé, transitions de pages fluides et animations "staggered" pour une navigation rythmée.

## Fonctionnalités Clés

- **Architecture Modulaire** : Code structuré en composants React réutilisables et scalables.
- **Navigation Dynamique** : Menu adaptatif (Desktop complet / Mobile Hamburger) avec une expérience cohérente sur tablette.
- **Performance & SEO** : Optimisation des balises meta, accessibilité et temps de chargement maîtrisés.
- **Interactivité Avancée** :
  - Barre de progression de lecture.
  - Système de "Scroll to Top" intelligent.
  - Effets de survol créatifs sur les grilles de médias.
  - Formulaires de contact optimisés pour la conversion.

## Stack Technique

- **Frontend** : React 19 + TypeScript
- **Styling** : Tailwind CSS 4 (Utility-first)
- **Animations** : Motion (Framer Motion)
- **Icons** : Lucide React
- **Routing** : React Router DOM

## Structure du Projet

```text
src/
├── components/         # Composants UI modulaires
│   ├── Navbar.tsx      # Navigation intelligente
│   ├── Hero.tsx        # Section d'impact
│   ├── Expertise.tsx   # Domaines d'intervention
│   └── ...             # Autres sections métier
├── lib/                # Utilitaires et helpers
├── App.tsx             # Configuration des routes et layout
└── main.tsx            # Point d'entrée de l'application
```

## Scalabilité

Le projet est conçu pour évoluer facilement :
- **Ajout de sections** : Structure de grille et composants de header standardisés.
- **Internationalisation** : Prêt pour l'intégration de bibliothèques i18n.
- **Backend** : Architecture compatible avec une intégration API (Express/Node.js).
